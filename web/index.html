Platform capabilities

Built for production LLM engineering

A complete evaluation toolkit engineered to catch regressions, test adversarial edge cases, and maintain performance SLAs.

Domain-Agnostic Engine
Point the evaluator at any document corpus — legal contracts, clinical guidelines, HR policies, financial filings, or tech docs via simple configuration.
Multi-Signal Scoring
Combines semantic cosine distance, token F1, numerical claim extraction, and contradiction checks to prevent false auto-passes.
Persistent & Incremental Index
Persistent ChromaDB with SHA-256 file hashing only re-indexes modified files. Supports paragraph, sentence, and fixed token window chunking.
Adversarial Robustness
Dedicated test suite for prompt injection, hallucination traps, misleading retrieval, negation traps, and missing context handling.
A/B Run Comparison
Compare prompt versions, model swaps, or chunking strategies side-by-side with per-category pass rate deltas and status change tables.
Structured Telemetry & CI
JSONL trace logging records per-phase durations (retrieval, generation, judge, attribution) for latency waterfall analytics and GitHub Actions gating.

Architecture

The evaluation loop

Every test query travels through a six-stage automated pipeline — from document retrieval to CI quality gating.

Step 01
Document Ingestion
Corpus text files loaded, hashed, chunked (paragraph/fixed token), and indexed in persistent ChromaDB.
Step 02
Vector Retrieval
Query embedded via all-MiniLM-L6-v2. Computes Hit Rate, MRR, nDCG@K, Precision@K, MAP, and Coverage.
Step 03
Multi-Provider Inference
Generates answer via configurable LLM client — supporting Groq (Llama 3.3), Ollama, OpenAI, or Anthropic.
Step 04
Multi-Signal & Ensemble Gate
Evaluates correctness via semantic sim, token F1, number matching, negation check, or weighted multi-model judge.
